Asincronía

Temporizadores

Los temporizados son útiles para añadir un tiempo de espera antes de ejecutar cualquier acción o para ejecutar una acción cada cierto tiempo. Reciben una funcion callback que es la que se ejecuta después del tiempo especificado o cada cierto tiempo especificado.

Asincronía en JavaScript

La asincronía en JavaScript funciona bajo un modelo LIFO (Last In First Out) sobre un solo hilo de ejecución:

Concurrencia y paralelismo:

Concurrencia

Ocurre cuando múltiples tareas progresan de manera simultánea, pero no necesariamente al mismo tiempo.

Ejemplo:

Si tienes un solo chef cocinando varios platos, este va cambiando entre tareas: pone a hervir agua, corta vegetales, y luego vuelve a revisar la olla. Solo se concentra en una tarea a la vez, pero está gestionando varias.

Paralelismo

Ocurre cuando múltiples tareas se ejecutan literalmente al mismo tiempo, usando varios recursos (como múltiples CPUs o núcleos de CPU).

Ejemplo

Si tienes varios chefs, cada uno cocinando un plato distinto al mismo tiempo, tienes paralelismo. Cada chef trabaja en su propia tarea sin interferir con los demás.

Callbacks

Primer mecanismo que utiliza JavaScript para trabajar la asincronía

No se recomienda utilizarlos porque se puede crear un callback hell

Son funciones que se envían como parámetros a una función, este parámetro/funciones se ejecuta inmediatamente se llama a la función principal

Promesas

Es otro de los métodos con el que se puede trabajar la asincronía, estas funcionan con dos recursos principales. result y reject

¿Qué método usar?

Si solo se va a hacer un proceso, se pueden utilizar las callbacks, si se requieren hacer más procesos asíncronos, se recomienda utilizar promesas
