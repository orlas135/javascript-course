Funciones en JS

Documentación sobre las funciones en JS

Funciones declaradas y Funciones expresadas

Las funciones declaradas son aquellas funciones que se definen con la palabra clave function, seguida del nombre al igual que los parámetros opcionales que deseemos utilizar en dicha función y, por último, las llaves que definirán el bloque de código que ejecutará dicha función
Para hacer uso de esta función, se debe hacer un llamado, especificando los argumentos (si así es requerido), de los que hará uso la función

Las funciones expresadas son aquellas que se pueden guardar en una variable o constante: No es obligatorio que tengan nombre

Funciones flecha

Las funciones flecha son una manera de crear funciones expresadas de una manera más abreviada

Si la sentencia que irá dentro de estas funciones flecha solo lleva una línea, se pueden omitir las llaves al igual que también se puede omitir la palabra return, ya que se convierte en algo implícito

Si se le da el nombre a una función expresada, se puede hacer referencia a ella misma dentro de la misma función

Arreglo (Arrays) en JS

Los arreglos arrays son una estructura de datos que permiten tener una colección de datos, sin importar su tipo, en una sola variable

Arrays

Documentación de los arreglos en JS

Los arreglos (Arrays) en JS son una manera de crear una colección de datos en una sola variable.


Esta estructura de datos permite operaciones de ordenamiento, búsqueda, actualización, creación y eliminación, como si de una base de datos se tratara
Los índices de los arreglos empiezan en cero y, otra de sus características, es que pueden almacenar en cada una de sus posiciones variables de distintos tipos de datos.

Objetos

Documentación sobre los objetos en JS

Los objetos son una colección de datos de tipo clave:valor que permite almacenar cualquier tipo de dato en cada una de sus propiedades. Estos tienen también la facultad de almacenar funciones como métodos.

Manejo de errores

Documentación del manejo de errres en JavaScript

Las sentencias try/catch permiten en Javascript capturar los posibles errores que se presenten al momento de ejecutar una sentencia de código. Si la ejecución tuvo éxito, el intérprete se irá directamente al bloque finally (si así se programó), o se saldrá del bloque try/catch para seguir la ejecucion del programa normalmente.
Estas sentencias, por consecuencia, hacen parte de las estructuras de control de JS

Sentencias break y continue

Las sentencias break y continue permiten controlar el flujo de una secuencia iterativa como lo son los ciclos
Si la condiciendo evaluativa se cumple, la palabra break, como su significado lo indica, interrumpirá y hará que se termine la secuencia iterativa. La palabra continue lo que hará es saltarse la iteración en la que la secuencia se encuentre en ese momento, lo que hará saltar a la siguiente iteración.

Destructuring

El destructuring o destructuración es una forma nueva de asignar valores de manera más ágil y dinámica a arreglos y a objetos.
Para hacer el destructuring, se debe de utilizar la misma sintaxis con la que se creó la variable "fuente", así que, si la variable fuente viene de un arreglo, la variable "destino" debe de crearse también en un arreglo, y si es de un objeto, como un objeto.
Para que la destructuración funcione en los objetos, la variable a crear debe llamarse igual que la propiedad/varaible "fuente"

Parámetros REST

Los parámetros REST son una forma de recibir parámetros en funciones o de asignar valores a las posiciones de un arreglo, sin saber exactamente cuantos valores/parámetros van a llegar como argumentos
